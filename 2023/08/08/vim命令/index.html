vim是什么？

vim是Linux环境下一款功能强大、高度可定制的文本编辑工具，类似于Windows下面的source insight、sublime、notepad。VI是visual interface的缩写，即可视化接口。vim 即 vi IMproved，在 vi 的基础上做了很多改进，如：增加了多级撤销、多窗口操作、崩溃后也可以恢复、增加了稳定性、关键字自动补全、上下文自动补全等功能。

vim的工作模式

vim一般有6种工作模式。

  • 普通模式：使用vim打开一个文件时默认模式，也叫命令模式，允许用户通过各种命令浏览代码、滚屏等操作。
  • 插入模式：也可以叫做编辑模式，在普通模式下敲击 i 、a 或 o 就进入插入模式，允许用户通过键盘输入、编辑。
  • 命令行模式：在普通模式下，先输入冒号:，接着输入命令，就可以通过配置命令对vim进行配置了，如改变颜色主题、显示行号等，这些配置命令也可以保存到/etc/vim/vimrc配置文件中，每次打开默认配置执行。
  • 可视化模式：在普通模式下敲击前盘上的 v 键，就进入可视化模式，然后移动光标就可以选中一块文本，常用来完成文本的复制、粘贴、删除等操作。
  • 替换模式：如果我们想修改某个字符，不需要先进入插入模式，删除，然后再输入新的字符，直接在普通模式下，敲击R键就可以直接替换。
  • EX模式：类似于命令行模式，可以一次运行多个命令

vim的各种工作模式可以通过不同的键进行切换，用户统一使用ESC键返回到普通模式。

在命令行中，可以使用”cp”命令来复制文件，使用”mv”命令来移动文件。

复制文件的命令格式是：
cp 源文件路径 目标文件路径

例如，要将文件a.txt复制到目标文件夹中，可以使用以下命令：
cp a.txt 目标文件夹路径

粘贴文件的命令是将文件从一个路径移动到另一个路径。在命令行中，可以使用”mv”命令来移动文件，这等效于将文件粘贴到新的位置。

移动文件的命令格式是：
mv 源文件路径 目标文件路径

例如，要将文件a.txt移动到目标文件夹中，可以使用以下命令：
mv a.txt 目标文件夹路径

请确保目标文件夹路径是存在的，并且有足够的权限来执行复制或粘贴操作。

使用git diff指令可以查看暂存区与当前版本之间的区别。

可以使用以下命令来查看暂存区与当前版本之间的差异：

git diff –staged

或者使用以下命令：

git diff –cached

查看工作目录中尚未添加到暂存区的修改：git diff命令不加参数时，会显示当前工作目录中所有修改了但尚未添加到暂存区的文件的差异内容。

比较暂存区与上一次提交的差异：通过命令git diff –staged或git diff –cached，可以比较当前暂存区与上一次提交的差异。

比较两次提交之间的差异：通过指定两个不同的commit或者branch，例如git diff commit1 commit2，可以比较两次提交之间的差异内容。

使用 git branch -d 命令可以删除一个已经合并到当前分支的分支。例如，如果想要删除名为 feature 的分支，可以运行以下命令：

git branch -d feature

如果要强制删除一个尚未合并的分支，可以使用 git branch -D 命令。例如，要删除名为 feature 的分支，可以运行以下命令：

git branch -D feature

请注意，删除一个分支将无法恢复其上的提交。在执行删除前，请确保您不再需要该分支上的任何更改。
