搭建博客的失败经历

Posted on 2023-08-08 In 编程 Views:
前言 一个个人博客，我其实早就想创建了但是，一来没有时间，二来不知从何入手，所以我一直没能创建。 今天的博客是基于hugo框架+github page 搭建的个人博客，已经能够访问，但是准备好的主题却没能正常显示。这就算是我折腾两天的结果吧！记录以未来怀念。 步骤 注意：我的步骤其实也是学习网络上hugo博客创建而来，觉得不清楚可以去搜索一下。 运行环境：win11 准备工具：1.git 2.hugo（下载最新版，记得下extended版的，下拉展开隐藏会有，我当时还找不到。。。） 安装hugo：你要先在文件路径没有空格的地方（推荐不是系统盘的根目录）创建hugo文件夹，在里面又创建一个bin文件夹和Sites文件夹，安装就是把这个就是把压缩包里的所有东西放到bin文件夹中（不要漏），把里面的.exe文件改名为hugo.exe，然后就是去配置hugo的环境变量了，配置为bin的路径，配置好就安装完成了。

接下来的操作：1.进入Sites文件夹打开cmd
2.hugo new site +网站名
3.cd 网站名
4.这一步，我们要去将一个别人做好的主题下载到我们的themes文件夹（有些版本的hugo好像是theme），方法有两个，一个是网站下载，一个是git下载。网站下载去hugo官网选好主题按download跳转到github界面，直接在
下载，然后解压到themes。如果是git下载，就要复制上面的ssh，在网站文件夹命令行 git init
git submodule add + ssh themes/主题名
echo theme = “主题名“ >> config.toml 应用主题
hugo new 文件夹名/文件名.md 添加.md内容，有些主题好像对文件夹名有要求
hugo server -t 主题名称 –buildDrafts 预览主题
也可直接运行博客
hugo server

在浏览器中访问 http：//localhost:1313/ 就可以看到博客预览的样子，按ctrl + C结束
接下来进入themes/主题名/examplesite中 复制config.toml到网站文件夹覆盖掉原来的config.toml配置文件。然后命令行 hugo 就能生成public文件夹了,生成之后在public文件夹中 git init
5.我们这里要到github上创建仓库，仓库名要求是：github用户名.github.io
把README也勾选上，是public仓库。
6.我们在生成的public目录下，为git本地库添加远程库（仅第一次需要）命令行是：git remote add origin +ssh
git pull origin master –allow-unrelated-histories
git add .
git commit -m “first commit” 这个first commit是提交说明，可以改成其他的
git push -u origin master 提交远程
<我看有些博客说在拉取前要先清空备份除.git文件夹以外的所有文件，add时再放入，但我没做好像也没事>
7.然后就能在github pages 那里看到。大体步骤就是这些，为啥搞了两天呢？
下面我来唠唠：
hugo文件夹作用：
·archetypes: Archetypes 是内容模板文件，其中包含预配置的首选项（日期、标题、草稿
等）。可以用自定义的预配置前端字段创建新的原型。
·assets: Assets 文件夹存储所有文件，这些文件由Hugo Pipes 处理（例如 CSS/Sass 文
件）。默认不创建这个目录。
·config.toml: Hugo使用 config.toml、config.yaml或 config.json（可以在网站根目
录中找到）作为默认网站配置文件。除了单独的配置文件之外，你还可以使用 config directory
来分隔不同的环境
·content：所有内容文件放在这里。顶级文件夹计为内容部分。如果你有 devops 和nodejs
部分，那么你需要有content/devops/first-post.md 和 content/nodejs/second-post.md 目
录。
·data：这里用来存储配置文件，Hug0会在生成你网站时用到
·layouts：以.html文件的形式存储模板。有关更多信息，请参见 Styling 部分
·static：存储所有静态内容：图片、CSS、JavaScript 等。当Hugo创建你的网站时，static 目
录中的所有资源均按原样复制
themes：你所选择的Hugo 主题
